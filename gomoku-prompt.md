## 一、项目概述

一款**中国风黑金配色**的五子棋网页游戏，单 HTML 文件，零依赖，浏览器直接打开即玩。

**核心体验：** 人机对战 + 双人对战，AI 秒回（10ms 级别），落子带三层中国风特效，胜利有游龙 + 印章动画。

**文件位置：** `/Users/kelvin/hermes/gomoku.html`

## 二、设计思路

### 2.1 设计哲学：盛唐国风「落子惊鸿」

整体走**黑底 + 鎏金**路线，灵感来自黑漆描金工艺：

| 层级 | 元素 | 设计 |
|------|------|------|
| 页面背景 | 纯黑 `#0d0d0d` | 隐现金色光晕 + 菱形回纹无缝贴图（80×80px Canvas 生成） |
| 棋盘 | 黑漆渐变 `#1a1814→#11100d` | 金色双层包边 + 金色网格线 + 金点星位 + 天元金莲暗纹 |
| 棋子 | 徽墨黑（隐紫光）/ 羊脂白玉（暖白半透明） | 带阴影、高光点 |
| 字体 | ZCOOL XiaoWei（毛笔行书） | 标题金色发光，正文衬线体 |
| 按钮 | 金色底 + 黑字 | 金边 + hover 金色光晕 |

### 2.2 落子特效（三层递进）

| 层次 | 时机 | 效果 |
|------|------|------|
| 预兆 | 悬停时 | 金色萤火光点，呼吸式明灭 |
| 落子 | 瞬间 | 棋子坠落动画 + 着点墨韵波纹扩散 |
| 余韵 | 之后 | 5 个金色粒子飘散上升 |

### 2.3 胜利特效

- 五连珠上画红色连线
- 连线上方金色游龙粒子流
- 棋盘中央浮现红色「胜」字印章（缩放 + 旋转动画）

### 2.4 音效（Web Audio API 合成）

- 落子：玉石轻碰声（高频短促）
- 胜利：编钟三连音（C5-E5-G5 上行）
- 悔棋：棋子滑过木面的轻擦声（白噪声滤波）

## 三、技术架构

### 3.1 整体架构

```
gomoku.html (单文件，约 1500 行)
├── <style>        CSS（CSS 变量配色、布局、动画、响应式）
├── <div cover>    封面层（全屏 overlay，点击后淡出）
├── <div game>     游戏层（标题、Canvas 棋盘、信息面板、按钮）
└── <script>       全部 JS 逻辑
     ├── 封面粒子生成
     ├── 金色回纹贴图生成（Canvas → data URL）
     ├── Polyfill（roundRect）
     ├── 棋盘绘制（drawBoard）
     ├── 棋子绘制（drawStone）
     ├── 特效系统（ripples / particles / winStamp）
     ├── 动画循环（requestAnimationFrame）
     ├── 游戏逻辑（placeStone / checkWin）
     ├── AI 引擎（Web Worker via Blob URL）
     ├── 音效（Web Audio API）
     └── UI 交互（click / touch / mode switch / undo）
```

### 3.2 数据流

```
用户点击 Canvas
  → handleClick / handleTouch
    → fromCanvasXY（坐标映射到交叉点）
      → placeStone(row, col)
        ├── 更新 board[][] 数组
        ├── 记录 moveHistory[]
        ├── 触发特效（ripples + particles + dropAnim）
        ├── checkWin() → winCells / 平局
        ├── 切换 currentPlayer
        └── 如果是 AI 回合 → aiMove()
                              → Web Worker 计算
                                → placeStone(row, col)
```

### 3.3 核心数据结构

```javascript
board[15][15]      // 0=空, 1=黑, 2=白
currentPlayer      // 1 或 2
gameMode           // 'pve' | 'pvp'
gameOver           // bool
moveHistory[]      // [{row, col, player}, ...]
lastMove           // {row, col} | null
winCells           // [[r,c], [r,c], ...] | null

// 特效状态
hoverPos           // {row, col} | null
dropAnim           // {row, col, player, t, startY, done} | null
ripples[]          // [{x, y, r, maxR, opacity, color}, ...]
particles[]        // [{x, y, vx, vy, life, maxLife, size, color}, ...]
winParticles[]     // 胜利粒子
winStamp           // {t, maxT} | null
```

### 3.4 AI 引擎

**算法：** Minimax + Alpha-Beta 剪枝 + 启发式评估

**运行方式：** Web Worker（Blob URL），不阻塞 UI 线程

**速度优化（关键）：**
| 策略 | 说明 |
|------|------|
| 候选半径 = 1 | 只搜索已有棋子周围 1 格，候选数 15-30 |
| 廉价排序 | 中心距离 + 邻域密度打分，绝不调用 evaluate() 排序 |
| 搜索深度 | 开局 2 层 → 前期 3 层 → 中期 3 层 → 后期 4 层 |
| 候选裁剪 | 深度 ≥ 3 时只保留前 12 个候选 |
| 快赢快防 | 每层先检查是否有五连珠，有则立即返回 |

**实测性能：** 中局 12 子 10ms，后期 36 子 11ms。
